  JWT Bearer token，格式：`Bearer YOUR_TOKEN`
</ParamField>

    <ResponseField name="user_preference" type="number">

      用于调整基于内容热度的推荐强度。值越大，越倾向于推荐热门内容。
    </ResponseField>

    <ResponseField name="diversity" type="number">

## 权重说明

推荐算法使用以下四个权重来计算推荐分数：

### 1. 内容相似度 (content_similarity)
- **作用**: 基于内容特征的相似性推荐
- **适用场景**: 用户看了某类视频，推荐相似主题的内容
- **调整建议**:
  - 提高：增强"看了这个的人也看了..."效果
  - 降低：减少重复类型内容推荐

### 2. 用户偏好 (user_preference)
- **作用**: 基于用户历史行为的个性化推荐
- **适用场景**: 根据用户点赞、收藏、观看时长等行为推荐
- **调整建议**:
  - 提高：更精准的个性化推荐
  - 降低：减少"信息茧房"效应

### 3. 热度 (popularity)
- **作用**: 基于内容受欢迎程度的推荐
- **适用场景**: 推荐热门、流行的内容
- **调整建议**:
  - 提高：更多推荐热门内容
  - 降低：给长尾内容更多曝光机会

### 4. 多样性 (diversity)
- **作用**: 增加推荐结果的多样性
- **适用场景**: 避免推荐内容过于单一
- **调整建议**:
  - 提高：推荐更多不同类型的内容
  - 降低：推荐更聚焦于特定类型

<Info>
  **权重平衡建议**：
  - 所有权重之和建议为 1.0（100%）
  - 典型配置：用户偏好 40%、内容相似度 30%、热度 20%、多样性 10%
  - 可根据业务目标灵活调整
</Info>

## 使用场景

此接口通常用于：
- 查看当前推荐策略配置
- 对比不同配置的效果
- 配置管理界面的数据展示
- A/B 测试中的参数记录
