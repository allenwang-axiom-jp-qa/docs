## 认证方式

所有客户端 API 都需要通过 JWT Token 进行认证。

### 请求头格式

```bash
curl -X GET "https://dev-client.beauty-666.com/api/v1/client/..." \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json"
```

### 参数说明

| Header | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | string | 是 | Bearer Token，格式：`Bearer {access_token}` |
| Content-Type | string | 是 | 固定值：`application/json` |

## 响应格式

### 成功响应

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    // 响应数据
  }
}
```

### 错误响应

```json
{
  "code": 400,
  "msg": "error message",
  "data": null
}
```

## 错误码说明

| 错误码 | HTTP Status | 说明 |
|--------|-------------|------|
| 0 | 200 | 请求成功 |
| 400 | 400 | 请求参数错误 |
| 401 | 401 | 未授权，Token 无效或已过期 |
| 403 | 403 | 禁止访问，权限不足 |
| 404 | 404 | 资源不存在 |
| 500 | 500 | 服务器内部错误 |
