---
title: '获取所有模块列表'
api: 'GET https://dev-admin.beauty-666.com/api/v1/oms/homepage/modules'
description: '获取所有首页模块，按 order 字段升序排列'
---

获取所有预设的 9 个首页模块列表，模块按 order 字段升序排列。系统预设模块不可新增或删除，只能编辑和启用/禁用。

## Headers

<ParamField header="Authorization" type="string" required>
  JWT Bearer token，格式：`Bearer YOUR_TOKEN`
</ParamField>

## Response

<ResponseField name="code" type="integer">
  响应状态码，0 表示成功
</ResponseField>

<ResponseField name="data" type="array">
  模块列表数组

  <Expandable title="模块对象字段">
    <ResponseField name="ID" type="integer">
      模块 ID
    </ResponseField>

    <ResponseField name="CreatedAt" type="string">
      创建时间（ISO 8601 格式）
    </ResponseField>

    <ResponseField name="UpdatedAt" type="string">
      更新时间（ISO 8601 格式）
    </ResponseField>

    <ResponseField name="key" type="string">
      模块唯一标识（系统预设，不可修改）

      可选值：banner、navigation、video_list、product_grid、swiper、icon_nav、notice、coupon、ad_banner
    </ResponseField>

    <ResponseField name="title" type="string">
      模块标题（可编辑）
    </ResponseField>

    <ResponseField name="type" type="string">
      模块类型（系统预设，不可修改）

      与 key 值相同，用于前端识别渲染组件类型
    </ResponseField>

    <ResponseField name="config" type="string">
      模块配置 JSON 字符串

      不同模块类型有不同的配置项，如：
      - Banner: `{"height":"200px","autoplay":true,"interval":3000}`
      - Navigation: `{"columns":4,"rows":2}`
      - Video: `{"layout":"grid","columns":2}`
    </ResponseField>

    <ResponseField name="order" type="integer">
      显示顺序，数值越小越靠前
    </ResponseField>

    <ResponseField name="enabled" type="boolean">
      是否启用，false 则前端不展示该模块
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="msg" type="string">
  响应消息
</ResponseField>

## 9个预设模块类型

<Info>
  系统预设了 9 个固定模块类型，不能新增或删除模块，只能编辑配置和启用/禁用。
</Info>

| 模块类型 | Key | 描述 | 子项数量限制 |
|---------|-----|------|------------|
| 轮播图 | banner | 首页主轮播图 | 2-6 |
| 快捷导航 | navigation | 图标导航 | 4-6 |
| 视频列表 | video_list | 推荐视频展示 | 4-8 |
| 商品网格 | product_grid | 商品展示网格 | 4-12 |
| 滑动卡片 | swiper | 横向滑动卡片 | 3-10 |
| 图标导航 | icon_nav | 小图标快捷入口 | 4-8 |
| 公告栏 | notice | 滚动公告 | 1-5 |
| 优惠券 | coupon | 优惠券展示 | 2-6 |
| 广告位 | ad_banner | 广告Banner | 1-3 |

## 使用场景

此接口用于：
- 管理后台获取所有首页模块列表
- 查看模块的启用状态和排序
- 为模块编辑功能提供数据源

## 注意事项

<Warning>
  - 模块总数固定为 9 个，不支持新增或删除
  - `key` 和 `type` 字段为系统预设，不可修改
  - 模块按 `order` 字段升序排列
  - 只有 `enabled=true` 的模块才会在前端显示
</Warning>

## 字段复用策略

不同模块类型使用相同的数据表结构，通过字段复用实现：

- **Banner模块**: title → 标题, image_url → 图片, link_url → 跳转链接, subtitle → 副标题/描述
- **Navigation模块**: title → 导航名称, image_url → 图标, link_url → 链接
- **Video模块**: title → 视频标题, subtitle → 时长/观看次数, image_url → 封面图, link_url → 视频链接
